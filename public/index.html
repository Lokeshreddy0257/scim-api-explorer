Identity Lifecycle Scenarios

Every scenario makes real SCIM API calls. Watch the HTTP traffic in the playground below.

🚀
01

Onboarding

New hire joins → AD group registered → user provisioned → Databricks workspace access granted automatically via SCIM.

POST/Groups
POST/Users
PATCH/Groups/<id> add member
🔄
02

Offboarding

Employee transfers teams → removed from old AD group → added to new group → access swapped in Databricks instantly.

PATCH/Groups/<id> remove
PATCH/Groups/<id> add
PATCH/Users/<id> dept
🚪
03

Termination

Employee exits → immediate lockout (active=false) → removed from all groups → account permanently deleted.

PATCH/Users/<id> deactivate
PATCH/Groups/<id> remove
DELETE/Users/<id>
📋
04

Recertification

90-day access review (SOC 2 / ISO 27001). Manager approves 3 accounts, revokes 1 stale account that hasn't been used in 95 days.

GET/Users review list
PATCH/Users/<id> revoke
PATCH/Groups/<id> remove
🌙
05

Nightly Verification

Automated job runs at 2 AM. Compares AD source of truth (expected) vs live Databricks state (actual). Detects drift — missing users, unauthorized group members — and auto-remediates via SCIM.

GET/Users + /Groups snapshot
DRIFT DETECTED
POST/Users re-provision
PATCH/Groups remove rogue

SCIM 2.0 Endpoints

All endpoints are live. Use the playground below to try them.

Users
GET/scim/v2/UsersList all users
POST/scim/v2/UsersCreate a user
GET/scim/v2/Users/<id>Get one user
PATCH/scim/v2/Users/<id>Partial update
PUT/scim/v2/Users/<id>Full replace
DELETE/scim/v2/Users/<id>Delete user
Groups
GET/scim/v2/GroupsList all groups
POST/scim/v2/GroupsCreate a group
GET/scim/v2/Groups/<id>Get one group
PATCH/scim/v2/Groups/<id>Add / remove members
DELETE/scim/v2/Groups/<id>Delete group
Discovery
GET/scim/v2/ServiceProviderConfigServer capabilities
🔑 All requests need: Authorization: Bearer demo-token-abc123
